# Notchi 1.1.0

A larger update that brings Codex into Notchi alongside Claude, adds richer mascot personalities, and makes usage and prompt context easier to trust.

## Codex Support

Notchi can now follow Codex CLI activity with the same kind of live presence previously built around Claude Code.

- Adds a Codex provider with hook installation, transcript parsing, and session lifecycle tracking
- Surfaces Codex prompts, tool activity, permission requests, compaction, aborted turns, and thread titles in the notch
- Shows Codex usage in the shared usage bar and previews provider-specific usage when hovering sessions

## Mascots And Emotion Analysis

The mascot system now has a broader personality range and can analyze prompts through more than one provider.

- Adds Claude and Codex sprite families with provider-specific styling
- Adds an elated emotion state and improves prompt emotion fallbacks
- Adds OpenAI as an emotion analysis provider with refreshed settings, testing controls, and an API key shortcut

## Notch Polish And Reliability

This release tightens the foundations that keep Notchi feeling responsive during active work.

- Introduces provider abstraction and a shared integration coordinator for multi-agent support
- Adds launch glow and sprite handoff polish, including a one-shot launch wave from the last-used mascot
- Improves Claude config directory detection, transcript-path handling, extra usage state, and Codex metadata stability
- Shimmers the working-indicator text and brightens the completed status for clearer at-a-glance state
