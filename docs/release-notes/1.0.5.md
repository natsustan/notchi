# Notchi 1.0.5

A patch release focused on cleaner session activity, steadier notch behavior, and more reliable Claude event handling.

## Session Activity Polish

The expanded panel is a little easier to follow during active Claude sessions, especially as replies grow and older sessions are dismissed.

- Auto-scrolls the activity feed when the panel opens and when assistant messages expand or collapse
- Renumbers visible sessions after dismissal so the list stays compact and intuitive
- Keeps the idle mascot visible in the collapsed header when no session is active

## Working State And Layout Fit

Notchi feels a bit more alive while Claude is working, and a bit more stable when it is waiting or reacting to clicks near the notch.

- Randomizes the working indicator verb each reply cycle to keep repeated runs from feeling repetitive
- Freezes the working symbol during waiting states so the status reads more clearly
- Improves click handling near the notch and recenters the notch layout

## Event Reliability And Safety

Claude session updates should now stay better aligned with what is actually happening in the transcript and socket layer.

- Skips synthetic transcript entries that should not appear in the activity feed
- Clears stale tool and assistant events when a new prompt starts
- Tightens socket permissions, ignores stale poll callbacks, and cleans up more reliably on quit
